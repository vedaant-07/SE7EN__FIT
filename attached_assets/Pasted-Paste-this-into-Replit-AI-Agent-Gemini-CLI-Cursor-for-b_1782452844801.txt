{

password,
role: "user"
}

Google login payload:
{
idToken,
role: "user"
}

Expected auth response:
{
token,
user: {
id,
name,
email,
phone,
role,
onboarding_completed,
avatar_url,
subscription_status
}
}

Store:

* JWT in SecureStore
* user object in secure/session store
* non-sensitive cache in AsyncStorage

Security:

* Never store password.
* Never expose JWT in logs.
* Mask sensitive error logs.
* Use Authorization: Bearer <token> for protected APIs.
* On 401, clear token and route to login.

Google login:

* Use native Google Sign-In for Android/iOS.
* Android OAuth client ID must be used for Android.
* iOS OAuth client ID must be used for iOS.
* Web client ID may be used as serverClientId.
* Backend verifies Google ID token.
* App receives backend JWT.
* Handle cancellation without showing scary error.
* Handle missing Play Services on Android.
* Handle network failure.
